# Brand Guidelines

These guidelines keep product docs, dashboard UI, and the public site aligned while the full visual identity is still forming.

## Name Casing

Use **Prontiq** as the written company and product name in prose, documentation, UI headings, legal references, and product references.

Use **prontiq** for developer surfaces where lowercase is conventional:

- Package names: `@prontiq/sdk`, `@prontiq/api`
- Domains: `prontiq.dev`, `api.prontiq.dev`, `docs.prontiq.dev`
- Code variables and namespaces: `const prontiq = new Prontiq(...)`, `prontiq.address.autocomplete(...)`
- Custom elements: `<prontiq-address>`

Use **prontiq** for the logo wordmark unless a future brand system deliberately changes it.

Do not use **ProntIQ**. The IQ idea is implicit in the name, but forced capitalization creates casing drift.

## Logo Direction

Use a lowercase **prontiq** wordmark direction for the logo. It should feel like a developer tool: sharp, modern, and quiet, not corporate.

When pairing the wordmark with a mark, the current public site uses a compact `P` monogram inside a small rounded square. Keep that direction until a formal logo system replaces it.

## Typography

Match the public site typography:

- Body/UI: `DM Sans`
- Editorial display accent: `Instrument Serif`
- Code/terminal/technical labels: `IoskeleyMono`

Use the serif face sparingly for contrast. The product should mostly read as a clean, technical interface.

## Color Cues

Use the public site palette as the baseline:

- Main background: `#06080d`
- Border / muted surface: `#1e293b`
- Primary text: `#e2e8f0`
- Primary accent: `#22d3ee`
- Supporting accents: `#818cf8`, `#34d399`, `#a78bfa`, `#fbbf24`

Avoid turning every surface cyan. Use cyan for active states, highlights, and the primary brand edge; keep most surfaces dark and restrained.

## Voice

Write plainly. Prefer concrete product language over hype.

Examples:

- Use: "Address validation powered by G-NAF."
- Avoid: "The ultimate AI-powered data platform for every workflow."
