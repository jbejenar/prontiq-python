---
title: "Validate"
description: "Verify any Australian address with confidence scoring."
---

The validate endpoint takes a full address string and returns the best matching G-NAF address with a confidence level.

## Basic usage

<CodeGroup>

```bash curl
curl "https://api.prontiq.dev/v1/address/validate?q=88+hill+street+broke+nsw+2330" \
  -H "X-Api-Key: YOUR_API_KEY"
```

```typescript Node.js (server-side)
const res = await fetch(
  "https://api.prontiq.dev/v1/address/validate?q=88+hill+street+broke+nsw+2330",
  { headers: { "X-Api-Key": "YOUR_API_KEY" } }
);
const { match, confidence } = await res.json();
```

</CodeGroup>

## Response

```json
{
  "match": {
    "id": "GANSW704227698",
    "addressLabel": "88 HILL STREET",
    "localityName": "BROKE",
    "state": "NSW",
    "postcode": "2330",
    "confidence": 1
  },
  "confidence": "high"
}
```

## Typo tolerance

Validate is fuzzy-matched — minor typos still resolve to the correct address with appropriate confidence:

```bash
# Both queries return the same match
curl ".../v1/address/validate?q=88+hill+street+broke+nsw+2330"
curl ".../v1/address/validate?q=88+hil+street+broke+nsw+2330"  # typo
```

A clean match returns `confidence: "high"`. A fuzzy match typically returns `medium` or `low` — use that signal to flag uncertain matches for review.

## Confidence levels

| Level | Score | Meaning |
|-------|-------|---------|
| `high` | > 20 | Strong match — address verified |
| `medium` | 10-20 | Partial match — review recommended |
| `low` | < 10 | Weak match — likely incorrect |
| `none` | — | No match found (`match` is `null`) |

## When to use validate vs autocomplete

| Scenario | Use |
|----------|-----|
| User is typing in real-time | **Autocomplete** — prefix matching, multiple suggestions |
| User submitted a complete address | **Validate** — full-text matching, single best match |
| Data cleaning / batch verification | **Validate** — one call per address |
| Building a search/typeahead UI | **Autocomplete** |
