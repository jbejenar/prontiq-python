## API Keys

Every request to Prontiq requires an API key in the `X-Api-Key` header.

```bash
curl "https://api.prontiq.dev/v1/address/autocomplete?q=test" \
  -H "X-Api-Key: pq_live_abc123..."
```

### Key formats

| Prefix     | Environment | Use                                 |
| ---------- | ----------- | ----------------------------------- |
| `pq_live_` | Production  | Real API calls in the live platform |
| `pq_test_` | Sandbox     | Testing, no billing, same data      |

### Getting a key

1. [Contact us](mailto:hello@prontiq.dev) to request access
2. We'll provision your API key and send it securely
3. Store it in environment variables — never commit to source control

<Warning>
  Never expose your API key in client-side code. Use a backend proxy to keep your key secret.
</Warning>

## Error responses

All errors follow a consistent JSON format with a `request_id` for tracing.

### 401 — Missing or invalid key

```json
{
  "error": {
    "code": "MISSING_API_KEY",
    "message": "Missing X-Api-Key header",
    "status": 401,
    "request_id": "req_abc123..."
  }
}
```

### 403 — Product not in plan

```json
{
  "error": {
    "code": "PRODUCT_NOT_ALLOWED",
    "message": "Product not included in your plan",
    "status": 403,
    "request_id": "req_abc123...",
    "details": {
      "product": "address",
      "allowed": ["address"]
    }
  }
}
```

### 429 — Quota exceeded

```json
{
  "error": {
    "code": "QUOTA_EXCEEDED",
    "message": "Monthly quota exceeded",
    "status": 429,
    "request_id": "req_abc123...",
    "details": {
      "product": "address",
      "used": 10000,
      "limit": 10000,
      "resets_at": "2026-05-01T00:00:00Z"
    }
  }
}
```

### 429 — Burst rate limited

```json
{
  "error": {
    "code": "RATE_LIMITED",
    "message": "Rate limit exceeded",
    "status": 429,
    "request_id": "req_abc123..."
  }
}
```

Burst-limited responses also include a `Retry-After` header.

## Request IDs

Every response includes an `X-Request-Id` header (format: `req_...`). Include this when contacting support — it traces directly to the log entry.

## Console/account endpoints use Clerk JWT, not API keys

A small number of dashboard-driven endpoints under `/v1/account/*` authenticate
with a Clerk session token instead of an API key. These are private console
routes, not public data-access endpoints, and they are not included in the
public API reference or SDK.

```bash
curl https://api.prontiq.dev/v1/address/autocomplete?q=test \
  -H "X-Api-Key: pq_live_..."
```

<Note>
  The current live platform still contains Stripe-linked account provisioning, but Prontiq's target
  commercial architecture is moving toward Lago. API key authentication for data endpoints does not
  change as part of that migration.
</Note>
